# Healthy You Closed Beta Tester Instructions

## What Healthy You Is

Healthy You is a wellness companion app for tracking routines, reviewing local wellness summaries, and testing a health assistant experience.

This beta focuses on core usability, local wellness tracking, account login, and safe beta behavior.

## Safety Disclaimer

Healthy You is not medical advice. It does not diagnose, treat, prevent, or cure medical conditions.

Do not use Healthy You for emergencies, urgent symptoms, medication decisions, or clinical decisions. Contact a qualified clinician or local emergency service if you need medical help.

## Install The APK

1. Download the APK from the approved closed beta distribution link.
2. On your Android device, allow installation from the trusted source if prompted.
3. Install the APK.
4. Open Healthy You.
5. If installation fails, capture the error message and report your device model and Android version.

Only install APKs shared through the official beta channel.

## Create Or Log In To An Account

1. Open Healthy You.
2. Choose register if you need a new beta test account.
3. Use an email address you are comfortable using for beta testing.
4. Create a password.
5. If you already have a beta account, use login.

Do not share your password in bug reports or screenshots.

## Screens To Test

Please try each main tab:

- Home/Data: daily briefing, weekly summary, analytics/report preview.
- Nutrition: local meal and hydration tracking.
- Fitness: local workout completion tracking.
- Schedule: local habit and medication daily tracking.
- Profile: settings, privacy/data controls, export preview, clear local wellness data.
- Medibot: text chat input and responses.

## Features Available In This Beta

- Account registration and login against staging backend.
- Local meal and hydration tracking.
- Local workout completion tracking.
- Local habit and medication status tracking.
- Local analytics and daily briefing.
- Profile settings and local privacy/data controls.
- Local export preview and local wellness reset.
- Medibot text chat with beta-safe fallback behavior.

## Features Intentionally Deferred

These are not expected to work in this beta:

- Food Scan.
- Voice input.
- File or image attachments.
- Push notifications and reminder delivery.
- Calendar integration.
- Backend account deletion.
- Live OpenAI-backed assistant responses while staging reports OpenAI is not configured.

Please report confusing copy around these items, but do not treat the deferred features themselves as bugs.

## What Not To Enter During Beta

Avoid entering:

- Highly sensitive medical records.
- Emergency symptoms or urgent medical information.
- Government ID numbers.
- Financial information.
- Passwords or secrets in Medibot chat.
- Information you would not want shown in a screenshot.

## How To Report Bugs

Use the beta feedback channel or issue form provided by the project owner.

Include:

- Device model.
- Android version.
- App build/version tag.
- Screen tested.
- Feature tested.
- Steps to reproduce.
- Expected result.
- Actual result.
- Screenshot or short video if safe to share.
- Severity and frequency.
- Network condition, such as Wi-Fi, mobile data, offline, or weak connection.

Use `docs/beta-feedback-template.md` as the structure if you are filing manually.

## Screenshots And Logs

If something breaks, helpful evidence includes:

- Screenshot of the error or unexpected screen.
- Short video of the steps, if easy.
- Approximate time the issue happened.
- Whether you were online or offline.
- Whether closing and reopening the app changed the behavior.

Do not share screenshots that expose private medical details, passwords, tokens, or other sensitive information.

## General Feedback

General feedback is welcome. Useful comments include:

- What felt confusing.
- What felt helpful.
- What you expected a button or screen to do.
- Any wording that made a deferred beta feature unclear.
- Any part of the app that felt too slow, fragile, or hard to trust.
